# 🚩 RedFlag – Fraud Detection Using SQL

## 📌 Project Overview

RedFlag is a SQL-based fraud detection project built on a simulated Indian digital payment dataset containing over **200,000 transactions**.
e
The objective of this projct is to identify suspicious transaction patterns using SQL queries without using machine learning. By applying SQL concepts such as aggregation, subqueries, Common Table Expressions (CTEs), and window functions, different types of financial fraud can be detected.

---

## 🎯 Project Objectives

- Detect suspicious transaction patterns using SQL.
- Apply real-world fraud detection techniques.
- Practice advanced SQL concepts on a large dataset.
- Understand how data analysts investigate financial fraud.

---

## 📊 Dataset

- Simulated Indian payment transactions
- 200,000+ transaction records
- Multiple users, merchants, cities, payment methods, and transaction types

---

## 🛠️ SQL Concepts Used

- SELECT, WHERE
- GROUP BY & HAVING
- Aggregate Functions
- CASE WHEN
- Subqueries
- Correlated Subqueries
- Common Table Expressions (CTEs)
- Window Functions
  - LAG()
  - ROW_NUMBER()
- DATE(), HOUR(), TIMESTAMPDIFF()

---

## 🚨 Fraud Patterns Detected

This project detects the following fraud patterns:

1. Velocity Fraud
2. Round-Amount Clustering
3. Card Testing
4. Failed-Then-Succeeded Transactions
5. Odd-Hour Concentration
6. Mule Accounts
7. Refund Abuse
8. Merchant Collusion
9. Just-Under-Threshold (Structuring)
10. Dormant-Then-Active Accounts
11. Velocity Spike
12. Geographic Impossibility
